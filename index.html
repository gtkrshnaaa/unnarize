1. Philosophy & Introduction

In an era where software complexity often neglects fundamental efficiency, Unnarize emerges as a philosophy. It does not aim to be a multipurpose language, but rather to force the machine to work in the most efficient, repetitive, and predictable manner possible.

Inspired by physics and thermodynamics, Unnarize is designed to indulge the flow of electricity itself. By directing instructions and data to consistently flow through the "same points over and over again," Unnarize creates an extreme form of locality of reference. This allows data and instructions to reside in the CPU's cache, the fastest memory closest to the processing core. Its core principle is: "Do something with one way, but perfect."

3. Language Guide

3.1 Basic Syntax

Unnarize uses a C-inspired syntax, making it easy to pick up.

Comments

Single-line comments start with //. All text following it until the end of the line is ignored.

// This is a comment.
var x = 10; // This is also a comment.

The `print` Statement

Use print to output values to the console. Each print call automatically adds a new line.

print("Hello, World!");
print(42);
print(3.14);

3.2 Variables & Data Types

Variables are declared using the var keyword. Unnarize uses dynamic typing, meaning a variable's type is determined by the value it holds and can change at runtime.

var message = "This is a string";
var number = 100;
message = 123; // Valid! The type of 'message' is now an integer.

Primitive Data Types

  • Integer: Whole numbers, e.g., -10, 0, 123.
  • Float: Decimal numbers, e.g., 3.14, -0.5.
  • String: Text enclosed in either " or '.
  • Boolean: Resulting from comparison operations. There are no literal true or false keywords; they are the result of expressions like 5 > 2.
  • Array: Dynamic lists created with array() function.
  • Map: Key-value collections created with map() function.
  • Module: Imported modules that contain variables and functions.
  • Future: Asynchronous computation results from async functions.

3.3 Operators

Arithmetic Operators

Unnarize supports standard arithmetic operators.

var a = 15;
var b = 4;
print(a + b); // 19
print(a - b); // 11
print(a * b); // 60
print(a / b); // 3 (integer division)
print(a % b); // 3 (modulo)

Comparison Operators

These operators produce a boolean value.

var result1 = 10 == 10;
print(result1); // true
var result2 = 10 != 5;
print(result2); // true
var result3 = 10 > 5;
print(result3); // true
var result4 = 10 < 5;
print(result4); // false

String Concatenation

The + operator is used to join strings. Other data types are automatically converted to their string representation.

var name = "Unnarize";
var version = "0.1";
print("Welcome to " + name + " v" + version);
// Output: Welcome to Unnarize v0.1

3.4 Control Flow

`if/else` Conditionals

var score = 85;
var passed = score >= 80;
if (passed) {
    print("Passed with good grades!");
} else {
    print("Needs more practice.");
}

`while` Loops

var i = 0;
var condition = i < 3;
while (condition) {
    print("Iteration #" + i);
    i = i + 1;
    condition = i < 3;
}

`for` Loops

for (var i = 0; i < 3; i = i + 1) {
    print("For count: " + i);
}

3.5 Functions

Functions are reusable blocks of code, defined with the function keyword.

function greet(name, age) {
    return "Hello, " + name + "! You are " + age + " years old.";
}

var greeting = greet("Kiann", "22");
print(greeting);

Recursion

Functions can call themselves, which is useful for solving problems like factorial calculation.

function factorial(n) {
    var condition = n <= 1;
    if (condition) {
        return 1;
    }
    var prev = factorial(n - 1);
    return n * prev;
}

var result = factorial(5);
print(result); // 120

3.6 Data Structures (Arrays & Maps)

Arrays

Arrays are ordered, dynamic lists of values.

var fruits = array();
push(fruits, "apple");
push(fruits, "banana");

var firstFruit = fruits[0];
print(firstFruit); // apple
fruits[1] = "mango";
var arrayLength = length(fruits);
print("Array length: " + arrayLength); // 2

var lastItem = pop(fruits);
print("Last item: " + lastItem); // mango

Maps

Maps are collections of key-value pairs. Keys can be Strings or Integers.

var user = map();
user["name"] = "Unnarize";
user["release"] = 2025;
user[1] = "One"; // Integer key

var userName = user["name"];
print(userName); // Unnarize
var hasRelease = has(user, "release");
print(hasRelease); // true

var deleted = delete(user, "release");
var stillHasRelease = has(user, "release");
print(stillHasRelease); // false

var allKeys = keys(user);
var firstKey = allKeys[0];
print("First key: " + firstKey);

3.7 Built-in Functions

Unnarize provides a comprehensive set of built-in functions that are implemented directly in the VM for maximum performance. These functions handle core data structures, async operations, and type conversions:

Array Functions

var arr = array();                    // Create new array (no arguments)
var newLength = push(arr, "item");    // Add item, returns new length
var item = pop(arr);                  // Remove and return last item
var size = length(arr);               // Get array length

// Array indexing and assignment
var firstItem = arr[0];               // Access by index
arr[1] = "new value";                 // Index assignment

Map Functions

var m = map();                        // Create new map (no arguments)
var exists = has(m, "key");           // Check if key exists (string or int)
var removed = delete(m, "key");       // Remove key, returns true if existed
var keyList = keys(m);                // Get array of all keys as strings
var size = length(m);                 // Get number of key-value pairs

// Map access and assignment
m["name"] = "John";                   // String key assignment
m[42] = "answer";                     // Integer key assignment
var value = m["name"];                // Access by key

Async Functions

// sleepAsync creates a Future that resolves after specified milliseconds
var future = sleepAsync(1000);        // Sleep for 1000ms, returns Future
var result = await future;            // Wait for Future to complete (returns 0)

// Async functions automatically wrap return values in Futures
async function delayedTask() {
    var delay = sleepAsync(500);
    await delay;
    return "Task completed";
}

var taskFuture = delayedTask();       // Returns Future immediately
var taskResult = await taskFuture;    // Wait and get result

String Operations

var text = "Hello";
var len = length(text);               // Get string length (built-in function)
var char = text[0];                   // Get character at index (single-char string)
var textLength = text.length;         // Alternative: property access for length

// String concatenation with + operator
var greeting = "Hello" + " " + "World";
var mixed = "Count: " + 42;           // Auto-converts numbers to strings

Type System & Conversions

// Automatic type conversions
var result = 3.14 + 2;                // Mixed int/float -> float result
var charCode = "A" + 0;               // Single-char string -> int (65)

// Type checking in conditionals
if ("non-empty") { print("truthy"); } // Non-empty strings are truthy
if (arr) { print("has items"); }      // Non-empty arrays are truthy
if (m) { print("has entries"); }      // Non-empty maps are truthy

4. Advanced Topics

4.1 Modularity (import)

Unnarize's module system uses file-based imports with environment isolation. Each module has its own scope and can export variables and functions to other modules.

Module Resolution

Unnarize searches for modules in this order:

  • UNNARIZE_PATH environment variable (colon-separated paths)
  • Project root recursive search
  • Module files must have .unna extension

Example Module

Create math.unna:

// File: math.unna - Module with variables and functions
var PI = 3.14159;
var E = 2.71828;

function add(a, b) {
    return a + b;
}

function multiply(a, b) {
    return a * b;
}

function factorial(n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

Using Modules

// File: main.unna - Import and use module
import math as m;

// Access module variables
var piValue = m.PI;
var eValue = m.E;
print("Constants: PI=" + piValue + ", E=" + eValue);

// Call module functions
var sum = m.add(10, 5);
var product = m.multiply(4, 7);
var fact = m.factorial(5);

print("Results: " + sum + ", " + product + ", " + fact);

Module Caching

Modules are cached by their logical name (not alias). Multiple imports of the same module share the same environment, ensuring consistency and performance.

4.2 Async/Await

Unnarize implements true asynchronous programming using pthread-based Futures. The VM creates separate threads for async operations while maintaining thread-safe coordination through mutex and condition variables.

Future Implementation

Unnarize's Future is a thread-safe container with:

  • Completion flag - tracks if operation is done
  • Result value - stores the resolved value
  • Mutex & condition variable - ensures thread safety
  • Blocking await - suspends execution until resolution

Functions declared with async automatically wrap their return values in already-resolved Futures, enabling seamless composition.

async function processTask(taskName, duration) {
    print("Starting task: " + taskName);
    
    // Simulate processing time using sleepAsync
    var delay = sleepAsync(duration);
    await delay;
    
    print("Completed task: " + taskName);
    return "Result of " + taskName;
}

async function runSimulation() {
    print("=== Async Simulation Started ===");
    
    // Start multiple async tasks
    var task1 = processTask("Database Backup", 2000);  // 2 seconds
    var task2 = processTask("File Processing", 1500);  // 1.5 seconds
    var task3 = processTask("Report Generation", 1000); // 1 second
    
    // Wait for all tasks to complete
    var result1 = await task1;
    var result2 = await task2;
    var result3 = await task3;
    
    print("All tasks completed!");
    print(result1);
    print(result2);
    print(result3);
}

runSimulation();

4.3 Native C Extensions (loadextern)

Unnarize's loadextern system allows loading compiled C shared libraries (.so files) for high-performance operations. This follows Unnarize's philosophy of delegation: let C handle computation while Unnarize orchestrates.

Library Resolution

Similar to modules, loadextern searches for libraries:

  • Explicit paths (if argument contains '/')
  • UNNARIZE_LIB_PATH environment variable
  • Project root recursive search

Libraries must export a registerFunctions(VM* vm) function that registers native functions using the VM's API.

Example: Math Library Plugin

Create math_lib.c:

// Example Unnarize external plugin: math_lib (self-contained)
// Build: gcc -shared -fPIC -o math_lib.so math_lib.c -lm -lpthread

#include 
#include 
#include 

// Mirror Unnarize's Value system (must match interpreter exactly)
typedef struct VM VM; // Opaque VM pointer

// Use interpreter's exact types for compatibility
typedef enum {
    VAL_INT, VAL_FLOAT, VAL_STRING, VAL_BOOL,
    VAL_MODULE, VAL_ARRAY, VAL_MAP, VAL_FUTURE
} ValueType;

typedef struct Value {
    ValueType type;
    union {
        int intVal;
        double floatVal;
        char* stringVal;
        int boolVal;        // Note: bool, not bool_storage
        void* moduleVal;
        void* arrayVal;
        void* mapVal;
        void* futureVal;
    };
} Value;

// Function signature for native functions (must match interpreter exactly)
typedef Value (*NativeFn)(VM*, Value* args, int argCount);

// VM registration function (dynamically resolved - exact name required)
void registerNativeFunction(VM* vm, const char* name, Value (*function)(VM*, Value* args, int argCount));

// Native function implementation
static Value mathPower(VM* vm, Value* args, int argCount) {
    (void)vm; // VM parameter unused in this function
    
    Value result;
    result.type = VAL_FLOAT;
    result.floatVal = 0.0;
    
    // Validate argument count
    if (argCount != 2) {
        return result; // Return 0.0 for invalid args
    }
    
    // Extract numeric values (supports int/float conversion)
    double base = 0.0, exponent = 0.0;
    
    if (args[0].type == VAL_INT) {
        base = (double)args[0].intVal;
    } else if (args[0].type == VAL_FLOAT) {
        base = args[0].floatVal;
    } else {
        return result; // Invalid type
    }
    
    if (args[1].type == VAL_INT) {
        exponent = (double)args[1].intVal;
    } else if (args[1].type == VAL_FLOAT) {
        exponent = args[1].floatVal;
    } else {
        return result; // Invalid type
    }
    
    result.floatVal = pow(base, exponent);
    return result;
}

static Value mathSqrt(VM* vm, Value* args, int argCount) {
    (void)vm;
    
    Value result;
    result.type = VAL_FLOAT;
    result.floatVal = 0.0;
    
    if (argCount != 1) return result;
    
    double value = 0.0;
    if (args[0].type == VAL_INT) {
        value = (double)args[0].intVal;
    } else if (args[0].type == VAL_FLOAT) {
        value = args[0].floatVal;
    } else {
        return result;
    }
    
    if (value < 0.0) return result; // Invalid for sqrt
    
    result.floatVal = sqrt(value);
    return result;
}

// Required entry point - called by Unnarize when loading library
void registerFunctions(VM* vm) {
    // Register multiple math functions
    registerNativeFunction(vm, "power", mathPower);
    registerNativeFunction(vm, "sqrt", mathSqrt);
}

Build and Usage

# Compile the plugin
gcc -shared -fPIC -o math_lib.so math_lib.c -lm -lpthread

Use the library in Unnarize:

// Load the compiled library
loadextern("math_lib.so");

// Use registered native functions
var powerResult = power(2, 8);        // 256.0
var sqrtResult = sqrt(16);            // 4.0

print("2^8 = " + powerResult);
print("√16 = " + sqrtResult);

// Native functions integrate seamlessly with Unnarize types
var numbers = array();
push(numbers, 4);
push(numbers, 9);
push(numbers, 16);

var i = 0;
while (i < length(numbers)) {
    var num = numbers[i];
    var root = sqrt(num);
    print("√" + num + " = " + root);
    i = i + 1;
}

Library Loading Process

When loadextern is called, Unnarize:

  • Resolves the library path using search rules
  • Loads the shared library with dlopen(RTLD_NOW | RTLD_GLOBAL)
  • Finds and calls the registerFunctions symbol
  • Stores the library handle for cleanup on VM shutdown
  • Maximum 256 external libraries can be loaded simultaneously

Error Handling & Edge Cases

Unnarize's interpreter includes robust error handling:

  • Division by zero - Caught for both int and float operations
  • Array bounds - Index out of range errors for arrays and strings
  • Map key errors - Missing key access throws errors
  • Type mismatches - Invalid operations between incompatible types
  • Call stack overflow - Maximum call depth protection
  • Memory allocation - Graceful handling of allocation failures
  • Future constraints - Cannot use Future in conditionals (must await first)

5. Examples & Usage Patterns

5.1 Case Study: Inventory Management

This comprehensive example demonstrates how all of Unnarize's core features work together to create a real-world mini-application. It showcases the language's strength in orchestrating data flow, managing state, and organizing code through modularity.

🎯 What This Example Teaches

Core Language Features:

  • • Maps for key-value inventory storage
  • • Arrays for ordered shipment data
  • • Functions for reusable logic
  • • Control flow with while loops and conditionals

Software Architecture:

  • • Modularity through import system
  • • Separation of concerns (data vs reporting)
  • • Clean interfaces between modules
  • • Error handling and edge cases

The example simulates a store inventory system where you track items, process incoming shipments, and generate detailed reports. It demonstrates Unnarize's philosophy: simple syntax, powerful composition.

Main file (main.unna):

import inventory as inv;
import report as rpt;

print("=== STORE INVENTORY DASHBOARD ===");

// Create and populate initial stock
var stock = inv.createInventory();
inv.addItem(stock, "apple", 10);
inv.addItem(stock, "banana", 5);
inv.addItem(stock, "chocolate", 12);

// Process a new shipment
var shipments = array();
push(shipments, 3); // apples
push(shipments, 5); // bananas
push(shipments, 2); // chocolates

var items = inv.itemKeys();
var idx = 0;
var itemsLength = length(items);
var condition = idx < itemsLength;
while (condition) {
    var currentItem = items[idx];
    var currentShipment = shipments[idx];
    inv.addItem(stock, currentItem, currentShipment);
    idx = idx + 1;
    condition = idx < itemsLength;
}

// Print the final report
rpt.printSummary("Unnarize Mart", stock);

Inventory module (inventory.unna):

// Inventory management module for Unnarize
// Provides functions to create and manage item stock using maps

// Create a new inventory (returns empty map)
function createInventory() {
    return map();
}

// Add items to inventory (or update existing quantity)
function addItem(inventory, itemName, quantity) {
    if (has(inventory, itemName)) {
        var currentQty = inventory[itemName];
        inventory[itemName] = currentQty + quantity;
    } else {
        inventory[itemName] = quantity;
    }
}

// Get quantity of specific item
function getQuantity(inventory, itemName) {
    if (has(inventory, itemName)) {
        return inventory[itemName];
    }
    return 0;
}

// Remove items from inventory
function removeItem(inventory, itemName, quantity) {
    if (has(inventory, itemName)) {
        var currentQty = inventory[itemName];
        var newQty = currentQty - quantity;
        if (newQty <= 0) {
            delete(inventory, itemName);
            return currentQty; // Return actual removed quantity
        } else {
            inventory[itemName] = newQty;
            return quantity;
        }
    }
    return 0; // Nothing removed
}

// Get all item names as array
function itemKeys() {
    var items = array();
    push(items, "apple");
    push(items, "banana");
    push(items, "chocolate");
    return items;
}

// Calculate total items across all types
function totalItems(inventory) {
    var total = 0;
    var itemList = keys(inventory);
    var i = 0;
    while (i < length(itemList)) {
        var itemName = itemList[i];
        total = total + inventory[itemName];
        i = i + 1;
    }
    return total;
}

// Check if inventory has sufficient stock
function hasStock(inventory, itemName, requiredQty) {
    if (has(inventory, itemName)) {
        return inventory[itemName] >= requiredQty;
    }
    return false;
}

Report module (report.unna) - Handles all output formatting and business logic analysis:

// Report generation module for inventory system
// Provides functions to generate and print inventory reports

// Print a detailed summary of the inventory
function printSummary(storeName, inventory) {
    print("");
    print("=== INVENTORY REPORT: " + storeName + " ===");
    print("");
    
    var itemList = keys(inventory);
    var totalItems = 0;
    var itemCount = length(itemList);
    
    if (itemCount == 0) {
        print("No items in inventory.");
        return;
    }
    
    print("Current Stock:");
    var i = 0;
    while (i < itemCount) {
        var itemName = itemList[i];
        var quantity = inventory[itemName];
        totalItems = totalItems + quantity;
        
        // Format item display
        var status = "";
        if (quantity < 5) {
            status = " (LOW STOCK)";
        } else if (quantity > 20) {
            status = " (OVERSTOCKED)";
        }
        
        print("  " + itemName + ": " + quantity + " units" + status);
        i = i + 1;
    }
    
    print("");
    print("Summary:");
    print("  Total item types: " + itemCount);
    print("  Total units: " + totalItems);
    
    // Inventory health assessment
    var avgStock = totalItems / itemCount;
    print("  Average stock per item: " + avgStock);
    
    if (totalItems < 20) {
        print("  Status: CRITICAL - Restock needed");
    } else if (totalItems < 50) {
        print("  Status: LOW - Monitor closely");
    } else {
        print("  Status: HEALTHY - Stock levels good");
    }
    
    print("=== END REPORT ===");
    print("");
}

// Generate a simple stock alert
function checkLowStock(inventory, threshold) {
    var itemList = keys(inventory);
    var lowStockItems = array();
    
    var i = 0;
    while (i < length(itemList)) {
        var itemName = itemList[i];
        var quantity = inventory[itemName];
        
        if (quantity < threshold) {
            push(lowStockItems, itemName);
        }
        i = i + 1;
    }
    
    if (length(lowStockItems) > 0) {
        print("LOW STOCK ALERT:");
        var j = 0;
        while (j < length(lowStockItems)) {
            var item = lowStockItems[j];
            print("  - " + item + " (" + inventory[item] + " units)");
            j = j + 1;
        }
    }
}

// Print top items by quantity
function printTopItems(inventory, count) {
    var itemList = keys(inventory);
    print("Top " + count + " items by stock:");
    
    // Simple display of first 'count' items (basic implementation)
    var i = 0;
    var displayed = 0;
    while (i < length(itemList)) {
        if (displayed >= count) {
            return;
        }
        var itemName = itemList[i];
        var quantity = inventory[itemName];
        print("  " + (displayed + 1) + ". " + itemName + ": " + quantity + " units");
        i = i + 1;
        displayed = displayed + 1;
    }
}

5.2 Pattern: Unnarize as a Glue Language

Unnarize excels as a glue language and orchestration tool. Its primary strength lies not in raw computational power, but in efficiently coordinating and connecting different components, especially high-performance C libraries.

Think of Unnarize as the conductor of an orchestra, it doesn't play every instrument, but it coordinates them perfectly to create harmony. When you need intensive computation, write it in C and let Unnarize orchestrate the workflow.

Design Philosophy: Orchestration over Computation

When facing a performance bottleneck, don't optimize the loop within Unnarize. Instead, delegate the heavy work to C. This pattern leverages Unnarize's strength as an orchestrator and C's strength as a high-performance worker.

Note: The functions processOneInC and processBatch in the examples below are conceptual, they represent C functions you would create and register using loadextern.

The less optimal pattern is calling a C function inside a Unnarize loop. This still incurs call overhead on every iteration.

// Less Optimal: Unnarize controls the loop
loadextern("processing_lib.so");

var results = array();
for (var i = 0; i < 1000000; i = i + 1) {
    var processed = processOneInC(i);
    push(results, processed); // 1 million function calls
}

The most optimal pattern is to delegate the entire loop to C in a single call.

// Most Optimal: C controls the loop, Unnarize orchestrates
loadextern("processing_lib.so");

// C will perform the 1 million iterations internally
var results = processBatch(1000000);
print("Batch processing completed: " + results);

// Now Unnarize orchestrates the workflow
var reportData = generateReport(results);
var success = saveToDatabase(reportData);
if (success) {
    print("Data saved successfully!");
}

Key Insight: Unnarize as Orchestration Language

Important: The functions like processOneInC, processBatch, generateReport, and saveToDatabase are conceptual examples. They represent the types of functions you can create as C extensions.

Unnarize's power lies in its ability to orchestrate complex workflows by connecting different specialized components:

  • Data Processing: Heavy computation in C libraries
  • I/O Operations: File reading, network requests, database operations
  • Business Logic: Workflow coordination, decision making, error handling
  • Integration: Connecting different systems and APIs

This makes Unnarize ideal for:

  • Data pipelines and ETL processes
  • System automation and scripting
  • API orchestration and microservice coordination
  • Scientific computing workflows
  • DevOps and deployment automation

Creating Your Own C Extensions

You can create any C function and make it available to Unnarize. Extensions are self-contained and use dynamic linking to interface with the interpreter. Here's how you would implement the conceptual processBatch function:

// File: processing_lib.c (self-contained plugin)
// Build: gcc -shared -fPIC -o processing_lib.so processing_lib.c -ldl

#include 
#include 

// Minimal mirror of interpreter public types
typedef struct VM VM; // opaque

// Use interpreter's exact types for compatibility
typedef enum {
    VAL_INT, VAL_FLOAT, VAL_STRING, VAL_BOOL,
    VAL_MODULE, VAL_ARRAY, VAL_MAP, VAL_FUTURE
} ValueType;

typedef struct Value {
    ValueType type;
    union {
        int intVal;
        double floatVal;
        char* stringVal;
        int boolVal;
        void* moduleVal;
        void* arrayVal;
        void* mapVal;
        void* futureVal;    // Future pointer for async operations
    };
} Value;

typedef Value (*NativeFn)(VM*, Value* args, int argCount);
typedef void (*RegisterFn)(VM*, const char* name, NativeFn function);

static Value processBatchInC(VM* vm, Value* args, int argCount) {
    (void)vm; // unused
    Value result; result.type = VAL_INT; result.intVal = 0;
    
    if (argCount != 1 || args[0].type != VAL_INT) {
        return result; // Return 0 on error
    }
    
    int count = args[0].intVal;
    if (count <= 0) return result;
    
    // Simulate intensive batch processing
    int sum = 0;
    for (int i = 1; i <= count; i++) {
        // Expensive computation delegated to C
        sum += i * i; // Example: sum of squares
    }
    
    result.intVal = sum;
    return result;
}

void registerFunctions(VM* vm) {
    RegisterFn regFn = (RegisterFn)dlsym(RTLD_DEFAULT, "registerNativeFunction");
    if (!regFn) {
        fprintf(stderr, "[plugin processing_lib] Failed to find registerNativeFunction\n");
        return;
    }
    regFn(vm, "processBatch", processBatchInC);
}

6. Conclusion & Key Insights

The Unnarize Paradigm

Unnarize represents a fundamental shift in how we think about programming languages. Rather than trying to be everything to everyone, Unnarize embraces focused excellence, it excels at orchestration, coordination, and creating harmony between different computational components.

When to Choose Unnarize

Consider Unnarize when you need:

  • Workflow Orchestration: Coordinating multiple systems, APIs, or processes
  • Performance-Critical Integration: Combining high-level logic with low-level C performance
  • Rapid Prototyping: Quick iteration on complex system interactions
  • Data Pipeline Management: ETL processes, data transformation workflows
  • System Automation: DevOps scripts, deployment orchestration

The Philosophy in Practice

Unnarize's design philosophy of "Do something with one way, but perfect" manifests in several key principles:

  • Locality of Reference: Code and data flow through predictable paths, maximizing CPU cache efficiency
  • Minimal Abstraction: Direct, transparent operations without hidden complexity
  • Composability: Small, focused components that work together seamlessly
  • Performance Delegation: Let each tool do what it does best, C for computation, Unnarize for coordination

Looking Forward

As software systems become increasingly complex and distributed, the need for effective orchestration languages becomes more critical. Unnarize positions itself not as a replacement for existing languages, but as a specialized tool for a specific purpose, creating efficient, maintainable, and performant system orchestration.

Whether you're building data pipelines, automating infrastructure, or creating hybrid applications that leverage both high-level logic and low-level performance, Unnarize provides the tools to create solutions that are both elegant and efficient.
